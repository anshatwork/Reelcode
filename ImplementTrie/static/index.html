Help

Add Route: Insert or update a route by providing a CIDR prefix (e.g., 10.0.0.0/8) and a next-hop label. The backend validates IPv4 CIDR format.

Lookup IP: Enter an IPv4 address to find the longest prefix match and its next hop. If no match exists, you'll see a not found message.

Routes Table: Displays all stored routes. Use the delete action to remove a route.

Notes:

  • Only IPv4 addresses/prefixes are supported.
  • Prefixes must be in CIDR notation, e.g., 192.168.0.0/16.
  • All changes are in-memory and reset on server restart.
